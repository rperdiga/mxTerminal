Concord 4.1.4

The headline of this release is always-loaded build rules for Claude Code. Concord now ships a project-level rules document that auto-loads into every Claude Code session running inside your Mendix project, alongside a user-owned folder for your own project-specific additions. The rules govern how the agent works (tool hierarchy, page-via-Maia doctrine, layout-first for branded apps, theme-module pattern, the named failure modes to guard against) — designed to make Claude Code materially better at building Mendix apps end-to-end without bailing on the hard parts.

This release also rolls forward everything since 1.1.1 — macOS support, the Maia bridge, bundled Mendix skill packs, default-on first-run wiring, an upgrade-apply path that takes care of itself, a port-leak fix, and the Mac-specific page-gen skill from 4.1.3.

What you'll experience

If you upgrade an existing install: open the Concord pane once after upgrading, and Concord wires itself up. The Concord MCP server starts, the bundled Mendix skill packs refresh, the new always-loaded rules install at <project>/.claude/rules/concord-build-rules.md, the user-owned .claude/rules/project/ folder is pre-created with a README, and a managed @-import block is added to <project>/CLAUDE.md so Claude Code loads the rules on every session. You'll see a short banner at the top of the pane explaining what just happened. Your existing terminal preferences (shell, theme, scrollback, etc.) are preserved verbatim.

If you're installing fresh: same flow, just without the "previous version" comparison. Drop Concord in, restart Studio Pro, open the pane — the wiring is automatic.

Cross-platform note

Concord ships on Windows and macOS. Most features are platform-identical; two differences worth calling out:

  • Maia integration is Windows-only — driving Maia from outside Studio Pro requires inspecting its WebView, which works on Windows (WebView2 + CDP) but isn't available on macOS (WKWebView requires host opt-in by Mendix). Concord ships a Mac-specific variant of the page-gen skill that hands the work to Maia through you instead of programmatically.
  • Hotkey-driven Studio Pro tools on Mac need Accessibility permission the first time — macOS prompts for it; you grant Studio Pro one-time access in System Settings → Privacy & Security → Accessibility.

Compatibility

  • Studio Pro: 11.10.0 or newer
  • Operating system: Windows 10 build 17763 (1809)+ · macOS 10.15 (Catalina)+
  • .NET runtime: 8.0 (bundled with Studio Pro 11)

Privacy

Concord is loopback-only — the Concord MCP server binds to 127.0.0.1; Maia integration uses Studio Pro's local WebView2 debug port. Nothing leaves your machine through Concord itself. The bundled skill packs are static Markdown files copied into your project. The CLI agents you run inside Concord follow their own privacy contracts.
